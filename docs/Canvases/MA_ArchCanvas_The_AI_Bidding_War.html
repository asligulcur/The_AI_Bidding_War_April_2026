Multi-Agent Architecture Canvas

94815 · Agentic Technologies · L6
The AI Bidding War — Phase 2 solution (April 2026): Hub-and-spoke procurement simulation — Buyer + three isolated vendors (A/B/C), Strategic Audit (agents/advisor.py), human gatekeeper, deterministic award_contract, optional governance nudges. Buyer broadcast confidentiality: same assistant text goes to every vendor as buyer_reply_prev; agents/buyer/agent.md + _PUBLIC_BROADCAST_BUYER_APPEND constrain cross-vendor leakage for standard scenarios; scenario_relax_public_broadcast defaults to relaxing that append for mode: malicious (see docs/Future Roadmap/PHASE_3_IMPLEMENTATION.md). Evaluation set: 11 scripted scenarios (config/scenarios.json, IDs 1–11). Measured results in evaluation/FINAL_EVALUATION_REPORT.md. Track A · Asli Gulcur.
1 Agent Design & Reasoning
1
🧠 Agent Model Strategy
Per Agent One Anthropic Claude model per ANTHROPIC_MODEL in .env (default Sonnet-class) for Buyer, Vendors A/B/C, Strategic Advisor (agents/advisor.py), and offline CFO narrative (scripts/cfo_monthly_report.py). Same API stack; roles differ by system prompts in agents/ (buyer, vendors, cfo_narrative) agent.md files, not by model ID.
Homogeneous Heterogeneous roles, homogeneous model family: simplifies operations and keeps negotiation dynamics attributable to isolation and personas rather than mixed model behavior. Malicious buyer uses the same stack with a different agent.md path when scenarios enable it.
Profiling Buyer (standard vs malicious), three vendor personas (premium / budget / mid-tier with distinct floors), Strategic Advisor audit prompt (ADVISOR_SYSTEM_PROMPT), offline CFO portfolio narrative (agents/cfo_narrative/agent.md + scripts/cfo_monthly_report.py → CFO Reports/) — executive-facing rollup of evidence logs, not in the live loop (Phase 2 §5.7). All Markdown personas + orchestrator-constructed user messages; no fine-tuning.
2
⚙️ Individual Behavior Implementation
Cognitive orchestrator.py drives discrete turns: per round, vendors A→B→C then Buyer — each step is a single LLM completion (not an unbounded ReAct loop). Proposed award triggers Strategic Audit (run_strategic_audit / _emit_strategic_audit), then human yes/no, then deterministic award_contract (no LLM on that path). If the Advisor API fails, orchestrator substitutes a logged Advisor error stub and still proceeds to human approval — traceable, not silent.
Spatial Anthropic API only. Each vendor thread is isolated: round 1 sees RFP + scenario framing + isolation footer; later rounds see only buyer_reply_prev (identical buyer assistant text for A/B/C), not other vendors’ raw bids. Buyer sees aggregated bid_summary (parsed BID_JSON). Public channel: buyer system prompt includes public-broadcast rules + runtime append for standard runs; malicious scenarios may omit append via scenario_relax_public_broadcast. At award time, format_negotiation_history stitches buyer hub + all three vendor threads for the Advisor — not shown to vendors during negotiation.
Temporal Fixed turn order within a round; up to three rounds per scenario; session ends on award, NO_AWARD, or round limit. Wall-clock bounded by API latency; no simulated calendar time between rounds.
3
⏱️ Temporal Dynamics & Simulation Model
Time model Discrete negotiation rounds — event-driven steps (each vendor reply, buyer reply, optional audit/gate) rather than continuous-time market simulation.
Sync Orchestrator is the barrier: vendors run sequentially in-process; buyer runs after bids are parsed; audit runs only after parseable AWARD_JSON.
Lag effects No incubation or delayed settlement — immediate logging to logs/scenario_*.log and logs/evidence_log_*.json after each step.
Horizon Single CLI session per orchestrator.py --id <N> with N ∈ 1–11 (Phase 2 scenario roster); batch or interactive acceptable; not real-time market hours.
2 Structure Implementation
4
🔀 Orchestration & Organizational Architecture
Pattern Hub-and-spoke: central orchestrator.py (control plane) invokes Buyer + three vendor workers; sequential vendor phase then buyer phase each round. Award path: audit → human → skill — not a debate or voting ensemble.
Static/Dyn Static roster (Buyer + Vendors A/B/C + on-demand Advisor); scenario parameters from config/scenarios.json. Phase 2 implements 11 scenario IDs (expanded from six Phase 1 archetypes) including HIPAA/GDPR, rip-and-replace, bid steering, conflict-of-interest, and other boundary cases — each with explicit governance vs negotiation baselines in evaluation/evaluation_rubric.md §2. No runtime agent spawning.
Sync Round boundaries; after human no, optional governance nudge steers the buyer’s next message; a new AWARD_JSON triggers re-audit before approval. Vendors are not recalled from the nudge gate alone — next round follows orchestrator rules (Phase 2 §5.4).
Adaptive Nudge path and revised AWARD_JSON / NO_AWARD; re-audit on revised awards — no new agent types spawned mid-session.
Finalize If no contract executed after the round loop, orchestrator.py runs a finalization segment: buyer must emit AWARD_JSON (in-budget) or explicit NO_AWARD. That path can repeat the same Strategic Audit → human approval → award_contract sequence as in-round awards (see diagram footnote in docs/System Architecture Diagram/ai_bidding_war_system_architecture.svg).
5
📡 Communication Infrastructure
Mechanism In-process Python: structured user/assistant messages to Anthropic; no separate message bus. Human input via terminal prompts at gate.
Schema Line-embedded BID_JSON: and AWARD_JSON: JSON; orchestrator parses; evidence serialized in evidence_log_*.json.
Protocol Custom application protocol; repo layout compatible with OpenClaw-style agent.md / skills/ packaging for external tooling — not MCP/A2A between agents in the live loop.
6
🗄️ Environment & Grounding Infrastructure
Per Agent LLM: Anthropic client only. Deterministic skills/award_contract.py for executed awards (writes logs/contract.txt when allowed). Vendors: no tools beyond text generation.
Shared In-memory orchestrator state: vendor_histories, conversation_buyer, buyer_reply_prev, parsed bid summaries — not a shared vector DB.
Denied Vendors cannot read each other’s threads or raw bids. They do see the same buyer prose; policy + runtime append reduce (not guarantee) restating others’ prices in that text. No live supplier APIs or email in v1.
Cold start Inputs at startup only: .env, config/scenarios.json, and per-role agent.md files under agents/ — not prior logs/ as control-plane inputs. During and after a run: session outputs include append-style scenario_*.log and evidence_log_*.json, plus optional contract.txt (overwritten on each successful award — not a multi-run history). These are outputs for humans and offline tools (harvest, CFO narrative); they do not reload into the orchestrator as cold-start state (aligns with system architecture diagram: control plane vs Zone 5 run outputs).
7
📐 Scale & Computational Architecture
Agent count Fixed: 1 Buyer + 3 Vendors + Strategic Advisor (invoked on award path); optional malicious_buyer persona swap per scenario — still one buyer instance.
Compute Local Python 3.10+ or Docker; LLM inference on Anthropic cloud; no elastic multi-node orchestration.
Mapping Single OS process; sequential API calls; parallelism not required for correctness (isolation is logical).
Constraints Three-round cap; hard budget ceiling enforced in Python (default $15,000); token usage per course/API limits. Harvest / export policy selects newest log per scenario ID; latency/stability not primary scored columns per FINAL_EVALUATION_REPORT.md §2.2 (Phase 2 plan).

3 Collective Behavior & Quality
8
🤝 Coordination Strategy & Mechanism
Strategy Role-based procurement: buyer competes for best offer; vendors do not coordinate. Evaluation uses two layers (Phase 2 / rubric): Governance — isolation, audit before award, guardrail, HITL traces; Negotiation — scenario-specific procurement goals. A run can governance-pass and negotiation-partial/fail and still be a valid evidence row.
Execution Turn-taking and parsing; Strategic Audit facilitates human oversight, not vendor–vendor consensus. Scoring references evaluation/test_cases.csv and FINAL_EVALUATION_REPORT.md §3.2, not chat prose alone.
Decision Buyer proposes award; human approves or vetoes execution; guardrail is dictatorial on price vs ceiling (blocks illegal spend even if prose claims success).
9
🔍 Emergent Behavior Detection & Steering
Conflict Competitive bids surfaced only to buyer as structured summary; no cross-vendor negotiation — reduces collusion by design. If buyer prose compares bids, all vendors may infer competitors’ numbers (mitigated for standard mode via public-broadcast rules).
Circuit-break award_contract rejects over-ceiling prices; max rounds; session termination in orchestrator. Advisor API failure → logged stub, flow continues to human (no silent skip of audit slot).
Herd check Separate threads and distinct personas limit uncritical agreement across vendors.
Drift Short horizon (3 rounds); audit compares proposal to RFP; human can veto.
10
✔️ Verification & Conflict Resolution
Evidence Transcripts in logs/; python scripts/export_evaluation.py runs harvest → rubric §3 fill → tables → failure log → syncs §3.2 and Appendix A in FINAL_EVALUATION_REPORT.md. Harvested CSVs (test_cases.csv, etc.) encode governance vs negotiation columns per Phase 2 methodology.
Separation Strategic Audit output is advisory text for the human; executable spend is only through award_contract after approval — LLM cannot unilaterally commit budget.
Contradiction Human veto yields NO_AWARD with logged rationale; over-ceiling blocked with ERROR string — no arbitration between vendors required.
Gaps NO_AWARD and negotiation partial/fail with governance pass are valid rows. Boundary / failure narratives: e.g. scenario 2 (guardrail stress), 3 (finance squeeze), 6 (compromised pressure), plus compliance-heavy IDs (7, 10, …) per §3.3 / Appendix A in the report.
11
🎯 Calibration & Validation Protocol
Data feeds Scripted scenarios in config/scenarios.json; logs as ground truth for harvest — not live procurement feeds.
Fit metrics Canonical criteria: FINAL_EVALUATION_REPORT.md §2 + evaluation/evaluation_rubric.md §2 per-scenario baselines. Harvest fields include Outcome_Reason, rounds, savings vs budget, feasibility band, Metrics_Audit, etc. (see harvest script docstring).
Frequency After new runs: python scripts/export_evaluation.py (or stepwise harvest / fill §3 / generate tables). Report metadata records evidence snapshot date; Phase 2 narrative §8 = plan, report = scored snapshot.
Benchmarks Course rubric and internal baselines; AgenticPay/Jannelli cited for methodology framing in related work — not external negotiation benchmarks.
4 State, Planning & Trade-Offs
12
💾 State Management & Feedback Architecture
Shared Orchestrator holds hub state; append-only logs on disk for replay and harvest.
Private Per-vendor message lists; buyer cumulative conversation; advisor does not retain state between invocations beyond the API call context.
Feedback Human rejection → optional nudge → buyer revises → new AWARD_JSON re-enters Strategic Audit before another approval attempt (Phase 2 gate sequence).
Lifecycle Session-scoped negotiation state (vendor_histories, conversation_buyer, buyer_reply_prev); no cross-session memory for buyer/vendors. Offline CFO narrative (CFO Reports/CFO_narrative_<YYYY-MM>.md) aggregates evidence_log_*.json by month for portfolio reporting — separate from in-run audit.
13
📋 Task Decomposition & Planning
Method Goal fixed per scenario ID (1–11): RFP, budgets, ceilings, modes (e.g. malicious buyer) from config/scenarios.json — not LLM-generated task DAG; bounded rounds substitute for open-ended planning.
Assignment Orchestrator assigns vendor/buyer/advisor roles by phase; buyer “plans” in natural language within rounds.
Assembly Parsed bids → buyer decision → optional award → audit + gate → contract file or abort; if rounds end without a contract, finalization buyer turn may repeat audit → gate → guardrail for a last AWARD_JSON / NO_AWARD.
Adaptive Nudge and revised buyer messages after veto; no dynamic scenario rewriting mid-run.
14
⚖️ Trade-Off Priorities
Priority 1 Governance and traceability (isolation, audit, HITL, deterministic spend) over maximizing deal count or model creativity.
Priority 2 Simulation fidelity (asymmetric info, structured evidence) over lowest API cost or sub-second latency — latency/stability not primary scored columns.
Accept LLM variance across runs; three-round cap; manual human gate; synthetic vendors; evaluation from harvested logs, not production ROI.
Reject Live vendor integrations, legal contract generation, dynamic agent spawning, unbounded negotiation horizon, cross-session buyer memory — explicit v1 out-of-scope.
