1 System Intent
1
🎯 Use Case / Purpose
Represent enterprise procurement in a simulated RFP: negotiate a 500-seat SaaS contract using consolidated bid_summary, and pursue value within policy. The same assistant reply is relayed to every vendor as the next user turn — treat it as a public channel: do not quote, compare, or rank other vendors’ prices or terms in that text (persona + runtime append). Thread isolation does not stop leakage if prose restates competitor bids.
2
✂️ Task Slice / Goal
Each round: consume orchestrator-built user messages (RFP, bid_summary JSON, budgets, ceiling). Respond with negotiation text; when ready to award, emit a parseable AWARD_JSON line. May CONTINUE across rounds until max rounds or stop conditions. Standard scenarios: buyer system prompt includes the Public broadcast section from agent.md plus the orchestrator’s _PUBLIC_BROADCAST_BUYER_APPEND.
3
👥 Users / Stakeholders
Operator Runs orchestrator.py — provides RFP text or accepts scenario default
Vendors Never addressed directly cross-thread; competition is mediated by hub
Human gate Approves or vetoes execution of award_contract after Strategic Audit
2 Bounded Behavior
4
⇄ Inputs / Outputs
IN conversation_buyer messages: scenario name, round index, full RFP, bid_summary (parsed BID_JSON for A/B/C), internal budget framing, hard ceiling text, award instructions.
OUT Buyer assistant text (stored as buyer_reply_prev and shown verbatim to vendors A, B, C in the next round); AWARD_JSON line when proposing an award; may state NO_AWARD or CONTINUE per session rules.
5
🔧 Tools / Permissions
ALLOW None in-vivo — orchestrator invokes award_contract only after human yes (listed in YAML skills for packaging compatibility).
DENY Shell, file write, external APIs, email, direct vendor-to-vendor contact.
6
🛡️ Autonomy / Human Checkpoints
Autonomous Natural-language negotiation strategy within rounds
Gated No contract execution without human approval after audit
Gated Deterministic guardrail rejects price > $15,000 regardless of buyer text
7
📜 Operating Rules / Policies
HARD Do not finalize spend in prose alone — AWARD_JSON triggers governed path
SOFT Hidden budget $15,000 annual — push back on over-quotes; align with scenario intent
ORG Persona from agent.md; model from ANTHROPIC_MODEL in .env; when scenario mode is not malicious, orchestrator may load agents/<BUYER_TYPE>/ (env, default buyer)
PUB Standard mode: scenario_relax_public_broadcast defaults false — runtime public-channel append is on. Override per scenario with relax_public_broadcast in config/scenarios.json.
Bounded Agentic System
3 Context & Evaluation
8
💾 Memory / Context
Session conversation_buyer — cumulative buyer-side thread only
Persist None across CLI sessions
Limit Bounded by max rounds and context window
Privacy Must not disclose hidden budget in ways that break scenario design. Public channel: assistant reply must not expose other vendors’ prices, ranks, or relative positions (prompt + runtime append — still best-effort LLM compliance).
9
⚠️ Failure Conditions / Escalations
Parse Missing or invalid AWARD_JSON when award intended — orchestrator may not enter award path
Policy Malicious scenario uses alternate buyer persona — not this file
Quality Suboptimal deal vs rubric negotiation criteria — still valid if governance passes
10
✅ Success Criteria
Role Credible procurement voice; parseable structured lines when awarding
Collective Feeds hub-and-spoke evaluation with other agents per scenario baselines
Safety No executed award above $15,000 — enforced in award_contract, not LLM. Confidentiality toward vendors: see Phase 3 doc for per-vendor messages / redaction backlog.
